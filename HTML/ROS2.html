understanding the concept

One of the primary ways for nodes to communicate in ROS2 is via topic. Here's a breakdown of the key concepts:

Topic

  • A topic is a named data channel that nodes use to communicate. Topics are identified by unique names.
  • Data flowing through a topic is accessible to any niode that subscribes to it, allowing multiple nodes to share and receive informationj on the same topics.

Message

  • A message is a single unit of data sent over a topic. Each message has a defined structure, known as a message type or msg, which determines the format and types of the data fields it can contain.
  • Examples of message types include "std_msgs/String" or "sensor_msgs/Temperature"

Publisher

A publisher is a node that sends data to a topic. When a node publishes a message, it writes this data to the topic, making it available for other nodes to read.

Subscriber

A subscriber is a node that reads the data from a topic. By subscribing to a topic, the node is notified whenever a new message is published, and it can then access and use the data.
